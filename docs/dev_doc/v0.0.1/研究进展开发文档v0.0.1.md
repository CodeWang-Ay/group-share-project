# 团队文献库 & 个人文献库功能需求开发文档
**版本号：V0.0.1**
**编写日期：2026-05-08**
**适用范围：智能计算实验室 - 研究进展模块开发**


### 一、基础核心数据（当前界面已有，可强化）
1.  **个人研究进展管理（学生端）**
    - 进度填报：支持按「周/双周/月度」更新进展，可上传附件（实验数据、论文草稿、代码链接）。
    - 目标管理：可设定短期/长期目标，自动关联组会汇报节点，逾期提醒。
    - 版本回溯：保留历史提交记录，支持对比不同阶段的进展变化。

2.  **团队研究进展概览（导师端）**
    - 进度看板：当前界面的「总人数/进度正常/滞后/未更新」统计，可按「博士生/硕士生/本科/年级/研究方向」筛选。
    - 风险预警：对进度滞后、长期未更新的学生自动标记，支持一键发起沟通提醒。



---

### 二、协作与沟通增强
1.  **针对性沟通功能**
    - 一对一进度沟通：当前界面的「沟通」按钮，可直接发起和该学生的私聊/留言，沟通记录自动归档到该学生的进展详情中。
    - 批量提醒：导师可一键给进度滞后的学生发送统一提醒，也可自定义提醒模板。

2.  **进度对比与趋势分析**
    - 个人进度趋势：学生可查看自己的进展更新频率、目标完成率，了解自己的科研节奏。
    - 团队进度对比：导师可按研究方向、年级对比不同成员的进度完成情况，快速识别瓶颈课题。

---

### 三、权限与流程控制
1.  **角色权限区分**
    - 学生：只能编辑自己的进展，可查看团队公开的进度看板，但无法修改他人数据。
    - 导师：可查看所有成员的进展，可发起反馈、标记里程碑、调整课题方向。
    - 管理员：可配置课题树、进度预警规则、提醒周期等系统设置。

2.  **提交规则配置**
    - 自定义提交周期：导师可设置不同学生的进展提交频率（如博士生每月一次，硕士生每两周一次）。
    - 强制提交与延期申请：逾期未提交自动提醒，学生可提交延期申请，导师审批后调整截止时间。

---

### 四、体验优化细节
1.  **数据可视化**
    - 个人进度甘特图：直观展示每个目标的时间线、完成状态。
    - 团队进度热力图：按研究方向展示整体进度分布，快速定位问题方向。

2.  **快捷操作**
    - 模板复用：学生可保存常用的进展汇报模板，每次提交直接套用，提高效率。
    - 智能填充：根据历史提交记录，自动推荐本次进展的撰写要点。
