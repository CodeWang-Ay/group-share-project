# 团队文献库 & 个人文献库功能需求开发文档
**版本号：V0.0.1**
**编写日期：2026-05-07**
**适用范围：智能计算实验室 - 文献库模块开发**

---

## 一、模块概述
本模块包含**团队文献库**与**个人文献库**两个独立子模块，核心目标是实现「团队共享协作」与「个人私有管理」的权限隔离，同时支持文献双向流转、删除权限控制、全局去重校验等核心能力，满足科研团队文献的协作与个人管理需求。

---

## 二、核心术语定义
| 术语 | 定义 |
| :--- | :--- |
| 团队文献库 | 团队内所有成员均可查看、检索、预览的公共文献池，数据为团队共享数据 |
| 个人文献库 | 仅用户本人可见、可管理的私有文献池，数据为用户私有数据，与团队库完全隔离 |
| 文献创建人 | 首次上传文献至对应库的用户，享有该文献的管理/删除权限 |
| 文献分享 | 以「拷贝副本」形式将文献从源库同步至目标库，源库数据不受任何影响 |
| 去重校验 | 通过多维度规则判断文献是否重复，避免同一文献在团队库中重复存在 |

---

## 三、功能需求详情

### 1. 权限与数据隔离规则
#### 1.1 数据隔离
- 团队文献库与个人文献库为两套独立数据，**底层物理隔离**，数据互不影响：
  - 个人库删除文献，不影响团队库数据；
  - 团队库删除文献，不影响用户个人库中已拷贝的副本。
- 两个库的统计数据（总文献数、新增数等）完全独立，切换页面时自动刷新对应统计结果。

#### 1.2 访问权限
- 团队文献库：所有团队成员均可查看、检索、预览全文，无访问限制。
- 个人文献库：仅文献归属用户本人可访问、管理，团队其他成员无任何访问权限（不可见、不可检索、不可预览）。

---

### 2. 文献新增/上传功能
#### 2.1 功能描述
用户可自主选择将文献上传至「团队文献库」或「个人文献库」，上传过程触发去重校验。

#### 2.2 业务规则
1.  **上传入口**：团队文献库与个人文献库均提供「添加文献」按钮，点击后弹出上传弹窗，支持选择上传目标库（弹窗默认选中当前页面所属库）。
2.  **去重校验逻辑**：
    - **团队库新增**：强制校验团队库内是否存在重复文献，重复则终止上传，弹窗提示「该文献已存在于团队文献库，禁止重复新增」；
    - **个人库新增**：不强制拦截重复，仅校验个人库内是否存在重复文献，重复则弹窗提示「该文献已存在于您的个人文献库」，用户可选择继续上传或终止。
3.  **文献归属绑定**：上传用户自动成为该文献的「创建责任人」，享有后续对应管理权限。

#### 2.3 输入/输出
- 输入：文献文件（支持PDF等格式）、标题、作者、DOI、关键词、年份、分类标签等元数据。
- 输出：上传成功后，对应库新增一条文献数据，统计看板同步更新。

---

### 3. 文献删除功能
#### 3.1 功能描述
用户可删除自己有权限的文献，不同库的删除权限与影响不同。

#### 3.2 业务规则
1.  **团队文献库删除规则**：
    - 普通用户：仅可删除**自己创建/上传**的团队文献，删除按钮对非本人上传的文献置灰/隐藏；
    - 团队管理员：额外拥有团队所有文献的删除权限（兜底管理能力）；
    - 删除影响：仅删除团队库内的源文献，不影响任何用户个人库中已分享的副本文献。
2.  **个人文献库删除规则**：
    - 用户可无限制删除个人库内所有文献；
    - 删除影响：仅删除个人库内的副本，不影响团队库的源文献数据。

#### 3.3 输入/输出
- 输入：用户点击「删除」按钮，触发删除操作；批量删除支持勾选多条文献批量操作。
- 输出：删除成功后，对应库文献列表移除该条数据，统计看板同步更新。

---

### 4. 文献跨库分享功能
#### 4.1 功能描述
支持文献在「团队文献库 ↔ 个人文献库」之间双向拷贝分享，以「副本」形式流转，源库数据保持不变。

#### 4.2 业务规则
1.  **团队文献 → 个人文献**：
    - 操作后：个人文献库新增一条该文献的独立副本，团队库数据无任何变化；
    - 校验逻辑：分享前校验个人库内是否存在重复文献，重复则提示「该文献已存在于您的个人文献库，无需重复分享」，终止分享。
2.  **个人文献 → 团队文献**：
    - 操作后：团队文献库新增一条该文献的独立副本，用户个人库数据无任何变化；
    - 校验逻辑：**强制校验团队库唯一性**，若团队库已有相同文献，直接提示「该文献已存在于团队文献库，禁止分享」，终止分享；
    - 归属规则：分享人自动成为该团队文献的创建责任人，享有对应删除权限。

#### 4.3 输入/输出
- 输入：用户点击「分享」按钮，选择目标库（团队/个人），触发分享操作；支持批量分享多条文献。
- 输出：分享成功后，目标库新增该文献副本，源库数据不变，统计看板同步更新。

---

### 5. 全场景去重校验规则
#### 5.1 触发场景
- 直接新增上传文献（团队库/个人库）；
- 个人文献分享至团队库；
- 团队文献分享至个人库。

#### 5.2 唯一性判定标准（优先级从高到低）
1.  文件MD5哈希：若上传/分享的文件MD5值与库内已有文件完全一致，判定为重复；

#### 5.3 去重控制策略
- 团队文献库：**强制全局唯一**，不允许存在重复文献，任何场景触发重复则直接拦截；
- 个人文献库：不强制全局唯一，仅做重复提示，用户可自主选择是否保留重复副本。

---

## 四、非功能需求
1.  **性能要求**：文献上传、分享、删除操作响应时间≤2s；去重校验接口响应时间≤1s；
2.  **数据一致性**：跨库分享、删除操作需保证事务完整性，避免出现数据不一致（如分享失败但目标库已新增数据）；
3.  **用户体验**：所有操作均需提供明确的加载状态、成功/失败提示，避免用户无感知等待；
4.  **权限安全**：后端需对所有删除、分享操作进行二次权限校验，防止前端绕过权限控制的非法操作。
